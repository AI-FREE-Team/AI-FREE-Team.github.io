Python - 實用開源套件


隨著網際網路的普及，html、Java、CSS 等程式語言日漸普及，成為跨領域學者必備的入門工具；同樣的劇本正複製上演在資料科學時代(大數據、人工智慧時代)，而引領主流的程式語言便是 Python，除了具備較低的進入門檻外，強大的開源社群更是讓 Python 完勝其他用來做資料科學的程式語言(如 R、Scala、SAS...等)。


Python 開源 - 巨人的肩膀

透過開源的工具，使用者在開發個人應用程式時，能在現成的框架下進一步開發，簡而言之，便是「站在巨人的肩膀上」，往往能使軟體開發工程師們，更省時、省力，也更能專注在開發個人的作品上。

今天要介紹的「Python - 實用開源套件」，便是由一位 Github 神人，Mahmoud Hashemi，所統整出來的一份Python開源清單；在分享這份開源清單前，容自由小編賣個關子，讓我們先看看神人分享這份清單的初衷。

無與倫比的 Python 應用程式

我們能從190多個已成功推出的 Python應用程式中學到什麼？

如果你正在閱讀此篇文章，表示你會讀、寫的程式語言，可能是 Python；對於你曾寫下的所有程式碼，可能多有些不完整之處。在令人為之興奮的軟體抽象化(名詞解釋：http://portablecontacts.net/wiki/computing/abstraction-software-engineering/)技術下，我們可以忽視一些重要的元素，使我們精心設計的模組、套件和開發框架轉變成現實世界中的軟體應用程式。

這便是為什麼我很高興地對外分享 Awesome Python Applications，一個由超過190多個項目所組成的開源套件列表，這些套件皆為：

(一) 線上、免費的開源知識庫資源。

(二) 大部分功能皆使用Python。

(三) 大多眾所周知，或在特定領域有相當利基點。

(四) 在相關平台上仍受維護或持續運作。

(五) 應用程式，而非函式庫或開發框架。

而這些列表中的套件，主要的目標對象皆不是開發人員(不必用pip或PyPI 即可安裝)，當然其中也有部分例外，但此份開源列表的廣泛程度相信是有目共睹的！

為什麼要花上幾週的的時間來統整這些開源的Python 應用程式呢？

除了歡慶假期(Christmas)外，有三大理由。

目標 # 1：更好的開發週期

自從我開始 談論 關於 Python的封裝，大家一直問我什麼封裝技術最適合他們的軟體。我一次又一次地感到訝異，在完成基本的交付條件之前，人們可以多深入地去開發應用程式。為了探索此議題，我找到了一個更基礎的問題：

為什麼有這麼多人從基本原理（blog文章和Stack Overflow）構建應用程式？

Python不是最有名的軟體之一嗎？不是有很多用Python撰寫成功的軟體實例嗎？您的應用程式有多大的獨特之處？

所以 無與倫比的Python應用程式 是一個真正嘗試針對那些困難艱澀的開發議題，而開闢新的流程。

撰寫應用程式時，先掃過此開源列表，以尋找最符合專案需求的項目，在透過這些應用程式作為回答問題的指南，將有助於解決開發環境建置、軟體部署、測試等抽象的議題。

回到學校，我從MediaWiki源代碼中學到了許多關於應用框架和軟體開發的知識，而非從任何課程中所學到的，也讓我持續受用至今。APA(引用研究論文格式) 將是實現與現實用戶的軟體開發全面教育的下一步。

簡而言之，每個出的應用程式可能寫出一千篇部落格的文章，但我們並沒有時間編寫它們。

目標＃2：更完整的Python開發循環

我們Python工程師也是軟體使用者；但與其他使用者不同，我們知道如何發現問題，甚至有可能對應用程式開發做出重大貢獻。

透過選擇Python軟體工具進行開發，我們都能向前邁進了一步，來幫助未來的軟體開發人員，能有什麼更好的方法可以進行開發？

我期望看到更多程式開發人員，在開發過程中與其他軟體做連結，但這些軟體是他們始料未及的 Python所撰寫。舉我對Twisted的貢獻來說，我便大量的使用了最喜歡的套件之一 Deluge，因此使用免費的套件可以開發出更多免費的軟體工具。

目標＃3：Python生態系統的基礎

隨著技術的演進，技術不斷地與時俱進，超越本身和最終用戶。基礎設施的硬體開發人員與軟體開發人員常失去連接，隨著時間的推移，軟體的開發能力也會變得較差，而此議題也常伴隨著軟體開發較少獲得開發者的重視；除非作者的技術非常傑出，否則大多數APA論文都僅有三位甚至兩位的星星數；而有些主要的Python應用程式也鮮少使用PyPI，因此下載統計資訊也無法幫助我們，即使他們使用PyPI，低階的函式庫也會有更多的分流；免費的軟體往往也較無法吸引大筆捐款、贊助，因此整體而言代表性也較欠佳。

這些應用程式不僅是免費的代表作，也是現存可用的Python 工具，它們不僅是Python實用與自豪的泉源，在精神上與實質上更需要我們的支持。我真心地希望 APA 能夠幫助 Python使用者社群在真實世界的軟體中，能夠有一席之地。

具體來說，是什麼意思？大家會注意到列表的結構，這不僅是一致性的資料呈現，更能作為嘗試獲取資料的API，我們也必須透過函式庫和應用程式之間的關係去探索我們的生態系。

我知道我在這裡談得很遠，指標不是一切，但是將Python FOSS (Free and Open-Source Softward；免費開源軟體)生態系統作為科學框架進行探索將會非常有趣。我們可以透過將函式庫視為“ 文章 ”並將應用程序視為“ 期刊 ”來獲得某種開發人員 h- index嗎？加入一些軟體用戶的使用經驗（通過社群 altmetrics和其他方法）可以讓我們更深切地了解軟體實際的影響力。
